## 🌐 Task 1: Deploy Backend to Render

### 1.1 Create Render Account
- [ ] Visit https://render.com
- [ ] Sign up with GitHub
- [ ] Authorize Render to access your repositories

### 1.2 Deploy Backend Service
- [ ] Click "New +" → "Web Service"
- [ ] Select `chatnalyxer` repository
- [ ] Configure:
  ```
  Name: chatnalyxer-backend
  Region: Singapore
  Branch: main
  Root Directory: chatnalyxer-backend
  Runtime: Python 3
  Build Command: pip install -r requirements.txt
  Start Command: uvicorn app.main:app --host 0.0.0.0 --port $PORT
  Instance Type: Free
  ```

### 1.3 Add Environment Variables
Click "Environment" tab and add:

- [ ] `DATABASE_URL` = `<your-supabase-connection-string>`
  - Get from Supabase dashboard → Settings → Database → Connection string
  - Format: `postgresql://user:pass@host:port/database`

- [ ] `SECRET_KEY` = (Auto-generated by Render) ✅

- [ ] `ALGORITHM` = `HS256`

- [ ] `ACCESS_TOKEN_EXPIRE_MINUTES` = `30`

- [ ] `OTP_EXPIRY_MINUTES` = `5`

- [ ] `OTP_MAX_ATTEMPTS` = `3`

- [ ] `RATE_LIMIT_REQUESTS` = `60`

- [ ] `RATE_LIMIT_WINDOW_MINUTES` = `60`

- [ ] `OTP_SERVICE_URL` = (Will be set after WhatsApp service is deployed)

### 1.4 Deploy
- [ ] Click "Create Web Service"
- [ ] Wait for deployment (5-10 minutes)
- [ ] Check logs for errors
- [ ] Note your backend URL: `https://chatnalyxer-backend.onrender.com`

### 1.5 Test Backend
- [ ] Visit: `https://chatnalyxer-backend.onrender.com/health`
- [ ] Should return: `{"status":"ok"}`
- [ ] Visit: `https://chatnalyxer-backend.onrender.com/docs`
- [ ] Should show API documentation

## 🐛 Troubleshooting

### Backend Deployment Failed
- [ ] Check build logs in Render dashboard
- [ ] Verify `requirements.txt` exists
- [ ] Ensure all dependencies are listed
- [ ] Check Python version compatibility

### WhatsApp Service Failed
- [ ] Check build logs
- [ ] Verify `package.json` exists
- [ ] Ensure Node version is compatible
- [ ] Check for missing dependencies

### Mobile App Can't Connect
- [ ] Verify backend URL in `config.ts`
- [ ] Check backend is running (visit `/health`)
- [ ] Ensure no typos in URL
- [ ] Check phone has internet connection

### OTP Not Working
- [ ] Verify WhatsApp service is running
- [ ] Check backend has correct `OTP_SERVICE_URL`
- [ ] Check WhatsApp service logs
- [ ] Ensure WhatsApp is linked (visit service UI)

## 💡 Tips

1. **Free Tier Sleep**: Render free tier services sleep after 15 min inactivity
   - First request after sleep takes ~30 seconds
   - Consider upgrading to paid tier ($7/month) for always-on

2. **Monitor Logs**: Check Render dashboard logs regularly
   - Backend logs show API requests
   - WhatsApp logs show OTP sending

3. **Database Backups**: Supabase auto-backs up on free tier
   - Can also export manually

4. **Version Control**: Always commit before deploying
   - Makes rollback easier if issues occur
